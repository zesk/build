## `buildContainer`

> Load Zesk Build in a preconfigured container

### Usage

    buildContainer [ image ]

Load Zesk Build in a preconfigured container
Starts in `/root/build`
Loads .STAGING.env first

### Arguments

- `image` - String. Optional. Image to load

### Return codes

- `0` - Success
- `1` - Environment error
- `2` - Argument error
