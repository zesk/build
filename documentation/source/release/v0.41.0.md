# Zesk Build release v0.41.0

`validate` updated in code everywhere. Any functions which look like `usageArgumentFoo` are now deprecated.

> Copyright &copy; 2026 Market Acumen, Inc.

- Previous version: v0.40.3
- Added deprecated updates to `catchReturn` for internal functions
- Renamed `environmentAddFile` to `buildEnvironmentAdd`
- All functions `usageArgument...` are deprecated and will be replaced by the deprecated binary - if you wish to keep
  using these, include `bin/build/tools/usage-deprecated.sh` to temporarily keep access to these. They will be removed
  in a future release.
- Removed all references to `# Usage:` in functions - the argument definition tends to be more accurate
- Added `shfmt` to pre-commit hook for bash (only if installed)
- Added some optimizations to `reloadChanges` prompt module to make it less slow
- Abandoned `buildFastFiles` and `sugar-fast.sh` - too soon to optimize this
- Added `timingElapsed`
- Removed `backgroundProcess` from core `bin/developer.sh` - need to test better for longer/slower processes
- Fixed `buildEnvironmentNames` and misspelled template variable `GASH_SOURCE`
- Fixed a bug in `listAppend` when adding strings whose ends match - would be a no-op
- Added `--version` to `_installRemotePackage` to enable fixed version installations for live stable environments
- Updated semantics of `version-function` and `url-function` in `_installRemotePath` to support new passed in requested
  version. Blank if latest is requested.
- `pathConfigure` and `pathRemove` now sanitize directory names to remove trailing slashes, helps avoid duplicates
- `bin/build/install-bin-build.sh --version v0.40.3` now works with the `v0.41.0` installer and greater
- `_installRemotePackage` supports usage via a pipe and avoids writing a file called `main` in this case
- Changed `which` usage to `command -v` but `command -v` caches binaries which have moved, so updated `whichExists` to
  check path and existence. `shopt -u checkhash` may fix this but do not want to worry about save/restore.
- Removed `BUILD_COLORS_MODE` and added `consoleConfigureDecorate` as a
  substitute ([Hey, look at when it was added!](./v0.7.6.md))
- Added `decorateInitialized` to better support correct setup for tools which configure their color mode
- `bashDocumentationExtract` now requires `sourceFile` and supports cached values in builds in the
  `bin/build/documentation/` directory when you add the flag `documentation-.
- Fixed the ordering of most documentation arguments so it's now `Type. Required.` `Type. Optional.`
- Added cached documentation to `bin/build/documentation` to speed this up as it never changes each build
    - See internal {SEE:buildDocumentationExtractionUpdate}
- Added some profiling tools in the identical directory: `profileFunctionHead` `profileFunctionMarker`
  `profileFunctionMarkerOthers` and `profileFunctionTail` - expects two local variables `$flag` and `$flags` to trigger
  the profiler
- Fixed an issue in `clearLine`; had a bug where it would prevent exiting if the internal function was interrupted
- `bashDebugInterruptFile` now outputs an error to `stderr` so it's clear that it's working. Errors look like:
