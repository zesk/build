# {applicationName} Security Policy

## Supported Versions

We support security patches for all versions listed [on our release page](./release/index.md) at the time of the
incident.

We will provide security patches as a new patch or minor version.

- Releases with major security issues are removed once the security issue is verified, marked within any repository
  library as invalid, and a new version is issued which replaces the issue and fixes the related security problems.
- Releases with minor security issues are marked within any repository library as risky, and a new version is issued
  which replaces the issue and fixes the related security problems.

No guarantees are made on timeliness of security patches.

For [dedicated support plans with a service level agreement, contact us](https://marketacumen.com/contact/?crcat=code&crsource=zesk-build&crcampaign=SECURITY.md&crkw=dedicated+support+plans+with+a+service+level+agreement,+contact+us).

## Reporting a Vulnerability

[Report vulnerabilities for {applicationName} on GitHub](https://github.com/zesk/build/issues/new).

Issues are checked working hours and you can expect a response within 8 business hours most weekdays. For urgent issues
use [our contact form](https://marketacumen.com/contact/?crcat=code&crsource=zesk-build&crcampaign=SECURITY.md&crkw=our+contact+form)
or [create a ticket](https://marketacumen.atlassian.net/servicedesk/customer/portals), tickets require registration and
email verification.

Please note **{applicationOwner}** does not guarantee response times or resolutions for non-clients at this time.
